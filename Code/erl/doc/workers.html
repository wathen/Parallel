workers - create and manage a pool of worker processes.

Description

workers - create and manage a pool of worker processes. Each worker process has a process state. The process state is an association list of tuples of the form {Key, Value}, where Key and Value are arbitrary Erlang terms When a process is created, this list is empty.

Worker process wait to receive tasks. The task is invoked on the current state and returns a new state.

broadcast/2

broadcast(W::worker_pool(), Task) -> ok

  • Task = (fun((ProcState::worker_state()) -> worker_state()) | fun((S::worker_state(), N::integer) -> worker_state()))

Each worker process performs Task. If

  • Task is a function with arity 1, then it is called with the current process state. The process state is updated to the return value of Task.
  • Task is a function with arity 2, then the first parameter is the current process state, and the second parameter is the index of the process. The process state is updated to the return value of Task. Process indices range from 1 to nworkers(W).
The value returned by Task becomes the new state for the process.

broadcast/3

broadcast(W::worker_pool(), Fun::Task, Args::List) -> ok

  • Task = (fun((ProcState::worker_state(), Arg) -> worker_state()) | fun((ProcState::worker_state(), N::integer, Arg) -> worker_state()))
  • length(Args) = workers:nworkers(W)

Each worker process performs Task, where Task is called with a per-worker argument. If

  • Task is a function with arity 2, then it is called with the current process state and lists:nth(N, Args), where N is the index of the process in W. Process indices range from 1 to nworkers(W).
  • Task is a function with arity 3, then the first parameter is the current process state, the second parameter is the index of the process, and the third parameter is lists:nth(N, Args).
The value returned by Task becomes the new state for the process.

get/3

get(ProcState, Key, Default::DefaultFn) -> term()

  • ProcState = worker_state()
  • Key = term()
  • DefaultFn = (undefined | fun(() -> term()) | fun((ProcState::worker_state()) -> term()) | fun((ProcState::worker_state(), Key::worker_state()) -> term()))

Get that value associated with Key in process state ProcState. Parameters:

  • ProcState: The state of the worker process.
  • Key: The key associated with the desired value.
  • DefaultFn: If S associates no value with Key then DefaultFn is used to determine the result of get(...).
Result: if there is a value associated with Key in S, then that value is returned. Otherwise, the return value is determined by Default:
  • If Default is an atom, then Default is returned.
  • If Default is a function with arity 0, then the return value is Default().
  • If Default is a function with arity 1, then the return value is Default(S).
  • If Default is a function with arity 3, then the return value is Default(S, Key).
  • If Default does not match any of these patterns, then an error is thrown.

random/2

random(M, ProcState0) -> {RandomValue, ProcState1}

  • M = number()
  • ProcState0 = worker_state()
  • RandomValue = number()
  • ProcState1 = worker_state()

Generate a random value. Parameters:

  • M: Specifies the range for random values ad described below
  • S0: The process state.
Result:
  • RandomValue
    • If is_integer(M), then RandomValue will be an integer uniformly chosen from 1..M.
    • If is_float(M), then RandomValue will be an float uniformly chosen in [0, M].
  • S1: The new process state.
    We update the random number generator state so that successive calls to random will produce different values.

  • If is_float(M), then RandomValue will be an float uniformly chosen in [0, M].
The state for the random-number generator is obtained from the worker process state, ProcState0, and the updated state (new value for the random number generator state) is returned in ProcState1.

retrieve/2

retrieve(W::worker_pool(), X) -> Values

  • X = (fun((ProcState::worker_state()) -> term()) | fun((ProcState::worker_state(), N::integer()) -> term()) | term())
  • Values = [term()]

Each worker evaluates Fun and the results are returned. The return value of retrieve(W, Fun, Args) is a list whose Nth element is the result of applying Fun in the Nth process. If

  • X is a function with arity 1, then it is called with the current process state.
  • X is a function with arity 2, then the first parameter is the current process state, and the second parameter is the index of the process.
  • Otherwise, X is taken as the key for looking up a value in the state of each worker process. If no match is found, then the atom 'undefined' is returned.

retrieve/3

retrieve(W::worker_pool(), Fun, Args::X) -> [Value]

Each worker evaluates Fun. X can be a list or a time-out specifier. If X is a list, then retrieve(W, Fun, X) is equivalent to retrieve(W, Fun, X, default_debug_timout()).
Otherwise, %% retrieve(W, Fun,X) is equivalent to retrieve(W, Fun, [ok, ok, ...], TimeOut}, where the list of ok's has the same length as W.

retrieve/4

retrieve(W::worker_pool(), Fun::What, Args::List, TimeOut) -> Values::List

Return a list of the results of evaluating What in each process of W.

If What is a function with arity 0, then it is called with no arguments (of course).
If What is a function with arity 1, then it is called with the current process state.
If What is a function with arity 2, then it is called with the current process state and lists:nth(N, Args), where N is the index of the process in W.
If What has arity 3, then the first parameter is the current process state, the second parameter is the index of the process, and the third parameter is lists:nth(N, Args).
If What is none of these, then we return the value of workers:get(ProcState, What, undefined) in each worker process.

If A worker process does not respond after TimeOut time units, then an error tuple is placed in the list. See set_debug_timeout/2 for the allowable values for TimeOut.

set_debug_timeout/2

set_debug_timeout(W, T) -> ok

  • W = worker_pool()
  • T = integer() | float() | default | infinity

Set the time-out for receive operations in the workers package to T. If T is an integer, it specifies the time-out in milliseconds.
If T is a float, it specifies the time-out in seconds.
If T is the atom 'default', the value from default_debug_timeout() is used.
If T is the atom 'infinity', then operations never time-out.

update/3

update(W::worker_pool(), Key::term(), X) -> ok

  • Fun = (fun((ProcState::worker_state()) -> term()) | fun((ProcState::worker_state(), N::integer()) -> term()) | [term()])
  • Values = [term()]

Each worker updates the value in its state (ProcState) associated with Key to the result of applying Fun. If there is no entry in the current state for Key, then an entry is created. If

  • X is a function with arity 1, then it is called with the current process state.
  • X is a function with arity 2, then it is called with the current process state and the index of the process.
  • X is a list, then length(X) must be the same as the number of workers in W (i.e. nworkers(W)). In this case, the value for Key in worker N is updated to the value of lists:nth(N, Args).
