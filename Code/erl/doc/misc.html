cut/2

cut(L, M) -> List_of_M_lists

Break L into M, roughly equal length segments. Parameters:

  • L: a list.
  • M: a positive integer -- the number of segments to create from L.
    Alternatively, M can be a non-empty list, in which case, length(M) is used as the number of segments to make.
Result:
  • List_of_M_lists: Each of the M segments has a length of length(L) div M or (length(L) div M) + 1.
Example:
  • cut([0, 1, 2, 3, 4], 3) -> [[0], [1,2], [3,4]]

See also: deal/2.

deal/2

deal(L, M) -> List_of_M_lists

Deal the elements of M around the M lists of List_of_M_lists. Parameters:

  • L: a list.
  • M: a positive integer -- the number dealt lists to create from L.
    Alternatively, M can be a non-empty list, in which case, length(M) is used as the number of new lists to make.
Result:
  • List_of_M_lists: The elements of L are 'dealt' in a round-robin fashion in to the M lists of List_of_M_lists.
    • hd(hd(List_of_M_lists)) = hd(L)
    • hd(lists:nth(I, List_of_M_lists)) = lists:nth(I, L) for I between 1 and M inclusive.
    • lists:nth((N div M)lists:nth((N rem M)+1, List_of_M_lists)) = lists:nth(N+1, L), for N between 0 and length(L)-1 inclusive.
Example:
  • deal([0, 1, 2, 3, 4], 3) -> [[0,3], [1,4], [2]]

See also: cut/2.

intervals/3

intervals(Lo, Hi, N) -> List

Divide [Lo, Hi] into N roughly intervals of roughly equal size.

  • Lo and Hi must be integers or floats.
  • N must be an integer.
  • List is a list of intervals. Each interval is a tuple, {Lo_i, Hi_i} where Lo_i is the lower bound for the ith interval, and Hi_i is the upper bound.
    • If Lo and Hi are both integers, then intervals in List have integer bounds.
    • If either Lo or Hi is a float, then intervals in List have floating point numbers for their bounds.
    • If either Lo or Hi is an integer whose magnitude is too large to represent as a float and the other is a float, then intervals(Lo, Hi, N) will produce a 'bad argument in an arithmetic expression' error.
Examples:
  • Hello World
  • misc:intervals(1,100,7) -> [{1,15},{15,29},{29,43},{43,57},{57,71},{71,85},{85,100}]
  • misc:intervals(1.0,100,7) -> [{1.00,15.1429},{15.1429,29.2857},{29.2857,43.4286},{43.4286,57.5714},{57.5714,71.7143},{71.7143,85.8571},{85.8571,100}]

logsteps/2

logsteps(Mant, Exp) -> [number()]

  • Mant = [number()]
  • Exp = [number()]

For each M in Mant and each E in Exp, M*10E is an element of the return list.
Example: logsteps([1,2,5], lists:seq(-1,2) -> [0.1, 0.2, 0.5, 1, 2, 5, 10, 20, 50, 100, 200, 500].
logsteps can be handy for generating a sequence of problem sizes for run-time measurements.

msg_dump/2

msg_dump(Who, Patterns) -> failed

  • Who = string() | nobody
  • Patterns = [string()]

Print the pending messages of a process -- for debugging. msg_dump prints an error message where each line is prefixed by the process pid and a colon. It states that a receive operation timed out, prints the list of patterns, and then prints all pending messages for the process.

  • If Who is a string, then the first line printed is
    "$PID: receive timed out in $Who"
    where $PID$ is the PID of this process, and $Who is the string from Who.
  • If Who is the atom 'nobody', then the first line printed is
    "$PID: receive timed out
    where $PID$ is the PID of this process.
Patterns is the list of strings that describe what receive was looking for.
msg_dump returns the atom 'failed'
Note: if the desired message were to arrive while we are printing, then it is possible that the pattern would be displayed, and later a message that matches that pattern would be printed.

pow/2

pow(N::number(), M::number()) -> number()

N raised to the Mth power.

  • If M is an integer, then NM is computed using the Russian Peasant algorithm with O(log N) multiplications. This produces an exact answer when N is an integer. Furthermore, it avoids an exception when N < 0.
  • Otherwise, M must be a float, in which case NM is computed using math:pow/2.
If N =0 and M < 0, then the atom 'undefined' is returned. If N < 0 and M is a float, then an arithmetic exception occurs. Likewise, there is an exception when M is a float and the NM produces a floating point overflow.

print/2

print(Term, N) -> ok

Print an erlang term on standard out. Unlike the erlang shell, print always prints a list of integers as a list of integers. Thus, print("hello world" is equivalent to io:format("~s", "[104,101,108,108,111,32,119,111,114,108,100]"). The point behind this is to prevent lists that really are supposed to be lists of integers from printing as strings just because every element of the list had a value in the range 32..127.
N gives (roughly) the number of terms to print. This means that if you ask to print a really big term, erlang won't be spewing stuff for several hours. If you want it to print the whole thing, use the atom 'infinity' for N.

sync/2

sync(P, TimeOut) -> ok | failed

  • P = pid()
  • TimeOut = integer() | infinity

Synchronize with process P. This process sends a 'sync' message to P, and P should call this function to send a 'sync' message to us. When we receive the message from P we continue. This ensures that both processes have made it to the point of their sync calls before either proceeds.

sync/3

Equivalent to sync(P, infinity, SndToken, RcvToken).

sync/4

sync(P, TimeOut, SndToken, RcvToken) -> ok | termR() | failed

  • P = pid()
  • TimeOut = integer() | infinity
  • SndToken = termS()
  • RcvToken = termR()

Synchronize with process P. This process sends SndToken to P, and P should respond by making a call to one of the versions of sync with this process as the target process.

  • If we receive RcvToken, we return it.
  • If we receive the default token, we return 'ok'.
  • If a timout occurs, we print a message dump and return 'failed'.
If SndToken is the empty tuple, {}, then the default msg:sync token is sent. Likewise, if RcvToken is the empty tuple, then we only look to receive the default msg:sync token.
A successful return from sync ensures that this process and process P have both reached the synchronization point before either proceeds.
