Description

wlog - Add event logging to some functions from the workers module. These functions behave like their counterparts from workers. However if workers:f(...) returns V, then wlog:f(...) returns {V, Log} where Log is an event-log (see module time_it) of "meaningful" events in the the execution of f. Generally, this refers to events involving inter-process interactions.

Like workers:reduce(W, Leaf, Combine, Root). If workers:reduce(W, Leaf, Combine, Root) returns V, then we return {V, Log}, where Log records the times of interactions between the various processes involved in the reduce operation.

retrieve/2

retrieve(W, Fun) -> {Values, Log}

  • W = worker_pool()
  • Fun = fun((ProcState::worker_state()) -> term())
  • Values = [term()]
  • Log = event_log()

Each worker evaluates Fun, and the results are assembled into the list Values. The times of key events are recorded in Log.
